        这里写一下原因

        • 交叉连接
          创建两个表的笛卡尔乘积，没有很大的意义
        • 自然连接
          链接只发生在两个表中有相同名字和数据类型的列上，如果名字相同数据类型不同，natural join会发生错误 自然链接也可以写成等值连接，算是等值连接的一种
        • 内连接
          其实就是查询两个表数据相等的列
        • 外连接
          除了能查到匹配的数据，还能查到不匹配的数据，用左外连接查询的是左表中的所有数据，右外连接查询的是右表中的所有数据，都查到就是全外连接，外连接也能用where子句来过滤数据 union all不会去重，union会去重

  • 索引介绍

    索引是对数据库表中的一列或多列值进行排序的一种结构，利用索引可以快速访问数据库表中的特定信息。索引是一种特殊的文件，他们包含对数据库的查询速度，mysql索引的建立对于mysql的高效运行是很重要的，索引可以大大提高mysql的搜索速度

    可能你有疑问，我加不加索引不都是从头到尾找数据吗，但实则如果不加索引，数据库的查找方式类似于顺序查找，而加了索引，数据库会使用B+树的数据结构来查找，一个是log2n，一个是n，所以加索引的效率要高得多，而每次增删改索引的时候树也随之变化，这也就是为什么索引增删改的开销会比较大

    • 优点

      • 通过创造唯一性索引，可以保证数据库表中每一行数据的唯一性
      • 加快搜索速度
      • 加快表与表之间的链接
      • 在使用分组和排序来进行检索的时候，可以减少查询中分组和排序的时间

    • 缺点

      • 创建索引和维护索引要耗费时间，这种时间随着数据量的增加而增加
      • 索引需要占用物理空间，数据量越大，占用空间越大
      • 会降低表增删改的效率，每次增删改索引都需要进行动态维护

    • 应用场景

      • 频繁作为查询条件的字段应该创建索引
      • 查询中排序的字段创建索引将大大提高排序速度
      • 查询中统计或者分组的字段

    • 非应用场景

      • 表中数据量很小的表
      • 频繁更新的字段
      • where条件用不到的字段
      • 经常增删改的字段

    • sql索引类型之普通索引

      最基本的索引，没有任何限制

    • sql索引类型之唯一索引

      索引列的值必须唯一，但允许有空值

    • sql索引类型之主键索引

      主键索引是一种特殊的唯一索引，不允许有空值

    • sql索引类型之组合索引

      组合索引是指在表的多个字段上创建一个索引，索引中包含了这多个字段的值，通过组合索引可以提高多字段查询的效率

      • 最左前缀原则

        组合索引的最左前缀原则是指在查询中使用组合索引时，必须按照索引中列的顺序进行查询，否则索引将不会被使用
        例如，组合索引index_name(column_name1,column_name2)，查询中必须先使用column_name1，然后才能使用column_name2，否则索引将不会被使用

  • MySQL事务介绍（TCL）

    事务是指作为单个逻辑单元执行的一系列操作，要么完全的执行，要么完全的不执行

    • 事务定义
      • 事务是一个最小的不可再分的工作单元；通常一个事务对应一个完整的业务
      • 一个完整的业务需要批量的DML语句来完成
      • 事务只和DML语句相关，或者说DML语句才有事务，这个和业务逻辑相关，业务逻辑不同DML语句的个数就不同
    • 事务的四个特征（ACID）
      • 原子性（Atomicity）

        事务是一个不可分割的工作单位，事务中的操作要么全部发生，要么全部不发生

      • 一致性（Consistency）

        事务必须使数据库从一个一致性状态变换到另一个一致性状态

      • 隔离性（Isolation）

        事务的隔离性是指一个事务的执行不能被其他事务干扰

      • 持久性（Durability）

        持久性是指一个事务一旦提交，它对数据库中数据的改变就应该是永久性的

    • 事务类型
      • 显式事务

        显式事务是指在SQL语句中使用BEGIN、COMMIT、ROLLBACK等语句来控制事务的开始、提交和回滚

      • 隐式事务

        隐式事务是指在SQL语句中没有使用BEGIN、COMMIT、ROLLBACK等语句来控制事务的开始、提交和回滚，而是依赖于数据库的默认行为

    • TCL语句
      语句 说明
      start transaction 开始一个事务
      commit 提交事务
      rollback 回滚事务

    回滚数据是必要的，因为事务的隔离性导致如果你写错了代码，直接断开链接，数据库会将你所改变的数据但没提交的数据锁起来不让其他事务访问，这会导致数据的不一致，所以回滚数据是必要的

    • 事务的并发问题

      • 脏读（Dirty Read）

        脏读是指一个事务读取了另一个事务未提交的数据，如果事务a读取到了事务b未提交的数据，而事务b回滚了，那么事务a读取到的数据就是脏数据

      • 不可重复读（Non-repeatable Read）

        不可重复读是指在一个事务中，对同一行数据进行多次读取，但是每次读取的结果都不同，这是因为在事务中，其他事务对数据进行了修改，导致事务读取到的数据不一致

      • 幻读（Phantom Read）

        幻读是指在一个事务中，对同一范围的数据进行多次读取，但是每次读取的结果都不同，这是因为在事务中，其他事务对数据进行了插入或删除，导致事务读取到的数据不一致

    • 事务的隔离级别

      • 读未提交（Read Uncommitted）

        读未提交是指一个事务可以读取到其他事务未提交的数据，这是最低的隔离级别，会导致脏读、不可重复读和幻读

      • 读已提交（Read Committed）

        读已提交是指一个事务只能读取到其他事务已提交的数据，这是默认的隔离级别，会导致不可重复读和幻读

      • 可重复读（Repeatable Read）

        在同一事务内，多次读取同一行数据的结果始终一致（即禁止不可重复读）。MySQL InnoDB 的默认级别

      • 串行化（Serializable）

        最高的隔离级别，强制事务串行执行，完全避免所有并发问题，但是一个接一个的执行，性能低
